# Flaky test quarantine ledger.
#
# Format:  <test-path>  <added:YYYY-MM-DD>  <review:YYYY-MM-DD>  <reason>
#
# A listed test is re-run (up to 3 attempts) by scripts/flaky-retry.sh, which
# both test runners go through, so a single unlucky roll no longer blocks a
# merge. It is NOT immunity: a deterministic failure fails all three attempts
# and still blocks. Every retry is reported in the CI job summary.
#
# The bar for adding an entry is deliberately high -- the non-determinism must
# be understood and must not be a defect in mutsu. Crashes, wrong answers, and
# anything whose root cause is unknown are never quarantined; they are bugs to
# fix. Read docs/flaky-test-policy.md before touching this file, and back every
# entry with numbers from scripts/ci-flake-survey.sh and scripts/flake-repro.sh.
#
# Entries expire: `make check-flaky-list` fails once the review date passes,
# which forces a re-measurement rather than a silent permanent quarantine.
#
# Keep sorted by path.
