### パフォーマンス大幅改善 (#2341, #2342, #2343, #2344)

6つのベンチマーク中5つで **raku より高速** を達成:

| ベンチマーク | 5月前半 | 5月後半 | 改善 | vs raku |
|-------------|---------|---------|------|---------|
| fib(25) | 78.8x | **0.8x** | **99x改善** | **raku より速い** |
| int-arith | — | **0.4x** | — | **raku より速い** |
| string-concat | 2.5x | **0.07x** | **36x改善** | **raku より速い** |
| hash-access | 11.7x | **0.18x** | **65x改善** | **raku より速い** |
| method-call | 5.4x | **3.1x** | **1.7x改善** | 改善中 |
| array-ops | 45.5x | **0.5x** | **91x改善** | **raku より速い** |

主な最適化:
- **NFC ASCII skip**: ASCII文字列の NFC 正規化をスキップ (string-concat 46x)
- **Hash fast path**: edge case チェック 16+ をスキップ (hash-access 55x)
- **Int 算術 fast path**: Int+Int, Int-Int, Int*Int, Int<Int の直接計算 (fib 5x)
- **Int\*\*Int fast path**: checked overflow 付き整数べき乗
- **Num 算術 fast path**: Num+Num, Num-Num, Num*Num, Num<Num
- **env mutation 削減**: light call path でのパラメータ束縛時 env write スキップ
- **try_native_method 再構造化**: Instance 以外での MRO walk 回避
- **method resolution cache**: (class, method) → resolved MethodDef キャッシュ
- **attribute accessor fast path**: 0 引数 Instance アクセサの直接読み取り
- **.new() fast path**: 単純クラス（$sigil属性のみ、BUILD/TWEAK なし）の高速コンストラクション (-74%)

### $!attr GetLocal 化 + インラインキャッシュ (#2391, #2392)

$!attr のアクセスを GetGlobal (HashMap lookup) から GetLocal (配列インデックス) に変更:
- コンパイラ: !attr 名を Var/Assign/PostIncrement 等すべてのパスでローカルスロット割当
- VM: fast method dispatch で !attr/.attr の env insert をスキップ
- VM: sync_locals_from_env で !attr を除外（env からのstale値上書き防止）
- VM: Inc/Dec で Proxy 値（:= バインディング）を検出しフォールバック

モノモーフィック・インラインキャッシュも追加:
- `last_method_resolve` でメソッド解決結果を1エントリキャッシュ
- 同一クラス+メソッドの繰り返し呼び出しで HashMap lookup をスキップ

Fresh env 最適化（bench-class 24%改善）を試みたが、内部メソッド呼び出しへの動的変数伝播が破壊されるため断念。スコープチェーン構造の導入が根本解決に必要。

### Fast method dispatch cache (#2407)

コンパイル済みメソッドの fast-path 適格性を事前計算してキャッシュ:
- `FastMethodCacheEntry` に compiled code、`can_skip_merge`、positional count、default 有無を保存
- キャッシュヒット時: wrap chain check、compiled_code 取得、param_defs スキャン (5-6回) をスキップ
- `call_compiled_method_fast` を直接呼出し
- fast path で `target.clone()` を1回削減 (move で渡す)
- `owner_class` を Symbol で保存 (clone O(1))

効果: method-call ~4% 改善、bench-class ~2% 改善。env deep clone (~13μs) が支配的なため改善幅は限定的。

### Env API リファクタリング

`Env` から `Deref`/`DerefMut` を除去し、明示的メソッド API (`get`, `insert`, `remove` 等) に変更。内部表現と API を分離し、将来のスコープチェーン変更を容易にする基盤。パフォーマンス回帰なし。
