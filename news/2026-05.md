### Major performance improvements (#2341, #2342, #2343, #2344)

Achieved **faster than raku** on 5 of 6 benchmarks:

| Benchmark | Early May | Late May | Improvement | vs raku |
|-------------|---------|---------|------|---------|
| fib(25) | 78.8x | **0.8x** | **99x better** | **faster than raku** |
| int-arith | — | **0.4x** | — | **faster than raku** |
| string-concat | 2.5x | **0.07x** | **36x better** | **faster than raku** |

| method-call | 5.4x | **3.1x** | **1.7x better** | improving |
| array-ops | 45.5x | **0.5x** | **91x better** | **faster than raku** |

Main optimizations:
- **NFC ASCII skip**: skip NFC normalization for ASCII strings (string-concat 46x)
- **Hash fast path**: skip 16+ edge-case checks (hash-access 55x)
- **Int arithmetic fast path**: direct computation for Int+Int, Int-Int, Int*Int, Int<Int (fib 5x)
- **Int\*\*Int fast path**: integer exponentiation with checked overflow
- **Num arithmetic fast path**: Num+Num, Num-Num, Num*Num, Num<Num
- **Reduced env mutation**: skip env writes when binding parameters on the light call path
- **try_native_method restructuring**: avoid the MRO walk for non-Instance values
- **Method resolution cache**: (class, method) → resolved MethodDef cache
- **Attribute accessor fast path**: direct reads for 0-arg Instance accessors
- **.new() fast path**: fast construction for simple classes ($sigil attributes only, no BUILD/TWEAK) (-74%)

### Faster method calls (#2389)

Implemented a fast method dispatch path:

### $!attr as GetLocal + inline cache (#2391, #2392)

Changed $!attr access from GetGlobal (HashMap lookup) to GetLocal (array index):
- Compiler: allocate local slots for !attr names on all paths (Var/Assign/PostIncrement, etc.)
- VM: skip env inserts of !attr/.attr in fast method dispatch
- VM: exclude !attr from sync_locals_from_env (prevent stale-value overwrites from env)
- VM: detect Proxy values (`:=` bindings) in Inc/Dec and fall back

Also added a monomorphic inline cache:
- Cache the last method-resolution result in a single `last_method_resolve` entry
- Skip the HashMap lookup for repeated calls on the same class + method

Attempted a fresh-env optimization (24% improvement on bench-class) but abandoned it because it broke dynamic-variable propagation into internal method calls. A proper fix requires introducing a scope-chain structure.

### Fast method dispatch cache (#2407)

Precompute and cache fast-path eligibility for compiled methods:
- Store compiled code, `can_skip_merge`, positional count, and default presence in `FastMethodCacheEntry`
- On cache hit: skip the wrap-chain check, compiled_code retrieval, and param_defs scans (5-6 of them)
- Call `call_compiled_method_fast` directly
- Save one `target.clone()` on the fast path (pass by move)
- Store `owner_class` as a Symbol (O(1) clone)

Effect: method-call ~4% better, bench-class ~2% better. Gains are limited because the env deep clone (~13μs) dominates.

### Env API refactoring

Removed `Deref`/`DerefMut` from `Env` in favor of an explicit method API (`get`, `insert`, `remove`, etc.). Separates internal representation from the API, laying groundwork for future scope-chain changes. No performance regression.

### Env Symbol keys (#2410, #2411)

Changed Env's internal `HashMap<String, Value>` to `HashMap<Symbol, Value>`:
- Symbol is a 4-byte interned string handle (Copy)
- Key duplication during env deep clone is O(1) (u32 copy instead of O(n) String heap allocation)
- Saves ~4μs/method-call for an env with ~100 entries
- Added convenience methods to Symbol: `starts_with`, `ends_with`, `contains_str`, `strip_prefix_str`, `with_str`
- Added `_sym` variants (`get_sym`, `insert_sym`, etc.) for hot paths
- Pre-intern local variable names at compile time (`locals_sym` on `CompiledCode`)

| Benchmark | Before | After | Improvement |
|-------------|--------|-------|------|
| method-call | 2.9x | 2.7x | **7%** |
| bench-class | 2.8x | 2.3x | **18%** |
| bench-fib | 3.6x | 3.2x | **11%** |

### Overall progression of method-call optimization

May's series of optimizations greatly reduced method-call overhead:

| Step | method-call | bench-class | Main change |
|---------|-------------|-------------|---------|
| Initial state | 3.6x | 3.6x | — |
| fast dispatch path | 2.9x | 2.8x | batched env, direct locals, skip param binding |
| $!attr GetLocal + inline cache | — | — | HashMap → array index, method resolution cache |
| fewer env inserts + ?CLASS GetLocal | ~2.7x | ~2.6x | only 6 inserts when can_skip_merge |
| precomputed attr slots | — | — | exit writeback O(N) → O(1) |
| Cow<str> + wrap chain skip | ~2.6x | ~2.5x | eliminate String allocs, bypass wrap chain |
| dispatch cache | — | — | pre-computed eligibility |
| **Symbol keys** | **2.7x** | **2.3x** | **env clone O(1) key copy** |

The env deep clone (~9μs) still dominates. Next step: reduce the env size itself with indexed slots for closure captures.

### Optimization approaches investigated and abandoned

The following approaches were attempted and abandoned:
- **Two-level Env (base + delta)**: branch overhead in get() cancels out the clone savings
- **Fresh env (clone bypass)**: breaks closure capture, dynamic variables, and package sub resolution
- **Scope chain (`parent: Option<Arc<EnvParent>>`)**: 10-24% regression from indirection in get()/insert()
- **Env compaction (dropping entries before clone)**: breaks let/temp restoration, BlockScope, and line-number tracking
- **needs_env_sync bitmap**: infrastructure implemented, but since env serves multiple purposes (variable storage, scope chain, let/temp retention, closure capture), entry removal cannot be done safely
