# FIX_FLOW.md - Roast test fixing workflow

## Goal

**roast テストを全部通すまで止めない。** これがこのプロジェクトの最終目標。

## 親プロセスの役割

あなたはただエージェントを起動して待つだけの存在ではない。
様々なアプローチを考えて、ズルせずにゴールに向かって進める **司令塔** である。

- エージェントが停滞していたら原因を分析し、介入する
- 途中までの成果があるなら、一旦そこまででPRを切る判断をする
- タスクが難しすぎるなら分割する、別アプローチを試す
- ビルドロック等の環境問題があれば回避策を考える
- 常に「今何が一番効率的か」を考えて行動する

## Steps

### 1. 現状の把握

```bash
./scripts/roast-history.sh
# tmp/roast-{panic,timeout,error,fail,pass}.txt が生成される
```

### 2. TODO.md の確認と差分チェック

```bash
git log --oneline --since="<TODO.md last update date>" --no-merges | head -50
```

TODO.md の blockers と quick wins から未実装項目をリストアップする。

### 3. スカウト+ワーカー方式によるタスク選定

#### 3a. スカウトエージェントでトリアージ (2-3分)

haiku モデルの軽量エージェントを起動し、失敗テストを高速トリアージする。
スカウトは **リサーチのみ** — コード変更は禁止。

スカウトへの指示:
- `tmp/roast-fail.txt` から各カテゴリ 8-10 テストを選び実行
- パス率 60% 以上のテストを特定
- 各テストについてパス率、根本原因、修正ファイル、難易度、影響数を報告

注意点:
- コンテナ/バインディングセマンティクスが必要なものは "hard" に分類
- 既に修正済み/進行中のテストはスキップリストに入れる

#### 3b. ワーカーエージェントに具体的な指示で起動

スカウトの結果を元に、各ワーカーに **具体的な修正指示** を与える:
- テストファイルパス、パス率、失敗サブテストの説明
- 根本原因と修正すべきファイル
- raku での期待動作の確認コマンド

#### 3c. フォールバック: 直接タスク割り当て

スカウトを使わない場合の優先順位:
1. **Panic** → 2. **Near-passing** → 3. **High-impact blockers**

### 4. エージェント起動

**必ず `isolation: "worktree"` を指定すること。**

各エージェントへの指示:
- `cargo build` → 修正 → `cargo clippy && cargo fmt` → `make test`
- feature branch → PR → `gh pr merge --auto --squash` → `gh pr checks --watch`

### 5. 完了したエージェントの即時補充

エージェントが完了したら、**確認せずに即座に**次のエージェントを起動する。
完了報告と新エージェント起動は同時に行う。常に3並列を維持すること。
ユーザーに「立てますか？」と聞くのは禁止。自分でタスクを選んで起動する。

### 6. 定期的な状況確認

10分ごとに各エージェントの進捗を確認し、ユーザーに報告する。

### 7. 停滞エージェントの対処

エージェントが停滞している場合（出力行数が増えない、同じコマンドをループ等）:
- 出力ログを読んで原因を分析する
- 必要に応じてエージェントを再起動（より具体的な指示で）
- 停滞の原因が根本的な問題（アーキテクチャ変更が必要等）の場合は、
  そのテストをスキップして別のタスクに切り替える

### 8. 進捗ファイルの定期更新

5個程度の PR がマージされるたびに、以下を自発的に実行する:

1. main を pull
2. `./scripts/roast-history.sh` を再実行
3. 新パステストがあればホワイトリストに追加してPR
4. TODO.md を更新してPR
5. HISTORY.tsv / HISTORY-pass.svg をコミット・プッシュしてPR

### 9. サイクルの繰り返し

全エージェント完了 → roast-history → 新パステスト確認 → 次のスカウト+ワーカーサイクル。
**roast テストが全部通るまでこのサイクルを回し続ける。**
